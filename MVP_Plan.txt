## 一、项目概述与 MVP 边界

### 1.1 核心验证假设

| # | 假设 | 度量方式 | 成功阈值 |
|---|------|----------|----------|
| H1 | "Mermaid + 双栏"阅读体验对极客/开发者有吸引力 | 页面平均停留时长、滚动深度 | 停留 > 2min，滚动 > 60% |
| H2 | 高质量 AI 架构图可通过 SEO 持续获取有机流量 | Google Search Console 点击量 | 上线 30 天 > 1000 次自然搜索点击 |
| H3 | 导出/私有库功能存在付费意愿 | Fake Door 邮箱收集转化率 | 点击→提交转化 > 8% |

### 1.2 MVP 明确不做

- ❌ 用户注册 / 登录体系
- ❌ 在线编辑器 / 用户 UGC
- ❌ 数据库 / 后端 API
- ❌ 真实支付流程
- ❌ 多语言国际化（MVP 仅中文，为目标市场聚焦）

### 选型决策记录

| 关注点 | 选择 | 理由 |
|--------|------|------|
| 框架 | **Next.js 14 App Router + TypeScript** | SSG 首屏极速，文件路由即目录结构，`generateStaticParams` 天然适配 MDX |
| UI | **Tailwind CSS + Shadcn UI** | Shadcn 非黑盒组件库（复制进项目即拥有），极客感强，可深度定制 |
| 图表 | **mermaid.js v11** | 行业标准的文本→图表引擎，支持 flowchart / sequence / class / state 等 12+ 图类型 |
| 画布交互 | **react-zoom-pan-pinch** | 体积仅 12KB gzip，API 极简，支持触屏双指手势 |
| 代码高亮 | **Shiki**（替代 Prism） | 与 VS Code 同引擎，支持 Mermaid 语法 TextMate Grammar，SSG 时预渲染无 FOUC |
| 内容 | **MDX + Frontmatter** | 纯文本 Git 管理，AI 可批量生成/校验，零数据库依赖 |
| 表单 | **Formspree（免费档）** | 50 提交/月免费，自带垃圾邮件过滤、Webhook、CSV 导出 |
| 部署 | **Vercel** | Next.js 原生支持，自动 Preview Deploy，全球 CDN |
| 分析 | **Vercel Web Analytics + 自定义事件** | 零配置接入，隐私友好（无 Cookie），自定义事件支持转化漏斗 |

```mdx
<Explain>
### 核心逻辑解析

**RAG（Retrieval-Augmented Generation）** 的本质是给大模型"开卷考试"：

1. **离线索引阶段**
   - **文档切分 (Chunking)**：将长文档按语义或固定窗口切为 256-512 token 的块
   - **向量化 (Embedding)**：通过 `text-embedding-3-small` 等模型转为高维向量
   - **入库存储**：向量存入 FAISS / Pinecone / Weaviate

2. **在线检索阶段**
   - 用户 Query 同样经过 Embedding
   - 在向量库中做 ANN（近似最近邻）检索，取 Top-K
   - 拼装为 Prompt Context，送入 LLM 生成最终回答

### 常见陷阱
- Chunk 过大 → 检索精度下降
- Chunk 过小 → 上下文碎片化，LLM 无法推理
- 未做 Re-ranking → Top-K 结果噪声多
</Explain>
```

### 4.1 Mermaid 渲染引擎

> **关键难点**：Mermaid 依赖 DOM，无法在 Node.js SSG 阶段运行，必须作为客户端组件处理。

```typescript
// components/graph/MermaidRenderer.tsx
'use client'

import { useEffect, useRef, useState, useCallback } from 'react'
import mermaid from 'mermaid'
import { useTheme } from 'next-themes'
import { getMermaidThemeConfig } from '@/lib/mermaid-themes'

interface MermaidRendererProps {
  code: string
  id: string
}

export function MermaidRenderer({ code, id }: MermaidRendererProps) {
  const containerRef = useRef<HTMLDivElement>(null)
  const [svgContent, setSvgContent] = useState<string>('')
  const [error, setError] = useState<string | null>(null)
  const [isRendering, setIsRendering] = useState(true)
  const { resolvedTheme } = useTheme()

  const render = useCallback(async () => {
    setIsRendering(true)
    setError(null)
    try {
      // 根据当前主题动态配置
      mermaid.initialize({
        startOnLoad: false,
        theme: 'base',
        themeVariables: getMermaidThemeConfig(resolvedTheme as 'light' | 'dark'),
        securityLevel: 'loose', // 允许 HTML 标签（如 emoji）
        fontFamily: 'Inter, "Noto Sans SC", monospace',
        flowchart: {
          useMaxWidth: false,  // 关闭自动宽度以支持缩放
          htmlLabels: true,
          curve: 'basis',
        },
      })

      const { svg } = await mermaid.render(`mermaid-${id}`, code)
      setSvgContent(svg)
    } catch (err) {
      console.error('Mermaid render error:', err)
      setError('图表渲染失败，请检查 Mermaid 语法')
    } finally {
      setIsRendering(false)
    }
  }, [code, id, resolvedTheme])

  useEffect(() => {
    render()
  }, [render])

  if (error) {
    return (
      <div className="flex items-center justify-center h-64 border border-destructive/50 rounded-lg bg-destructive/5">
        <div className="text-center">
          <p className="text-destructive font-mono text-sm">{error}</p>
          <button onClick={render} className="mt-2 text-xs underline">
            重试
          </button>
        </div>
      </div>
    )
  }

  if (isRendering) {
    return (
      <div className="flex items-center justify-center h-64">
        <div className="animate-pulse text-muted-foreground">渲染中…</div>
      </div>
    )
  }

  return (
    <div
      ref={containerRef}
      className="mermaid-container"
      dangerouslySetInnerHTML={{ __html: svgContent }}
    />
  )
}
```

```typescript
export function WaitlistModal({
  open,
  onOpenChange,
  feature,
  title,
  description,
  slug,
}: WaitlistModalProps) {
  const [email, setEmail] = useState('')
  const [state, setState] = useState<SubmitState>('idle')

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault()
    if (!email || !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) return

    try {

        setState('error')
      }
    } catch {
      setState('error')
    }
  }

  const handleClose = (isOpen: boolean) => {
    if (!isOpen) {
      // 关闭时重置状态
      setTimeout(() => {
        setState('idle')
        setEmail('')
      }, 300)
    }
    onOpenChange(isOpen)
  }

    <Dialog open={open} onOpenChange={handleClose}>
```

### 4.6 内容解析引擎

```typescript
// lib/content.ts

import fs from 'fs'
import path from 'path'
import matter from 'gray-matter'
import type { GraphFrontmatter, GraphMeta, CategoryInfo } from './types'

const CONTENT_DIR = path.join(process.cwd(), 'content')

/**
 * 递归扫描 /content 下所有 .mdx 文件
 */
export function getAllGraphs(): GraphMeta[] {
  const graphs: GraphMeta[] = []

  function scan(dir: string, relativePath = '') {
    const entries = fs.readdirSync(dir, { withFileTypes: true })
    for (const entry of entries) {
      const fullPath = path.join(dir, entry.name)
      const relPath = path.join(relativePath, entry.name)

      if (entry.isDirectory()) {
        scan(fullPath, relPath)
      } else if (entry.name.endsWith('.mdx')) {
        const raw = fs.readFileSync(fullPath, 'utf-8')
        const { data } = matter(raw)
        const fm = data as GraphFrontmatter
        const wordCount = raw.length
        graphs.push({
          ...fm,
          filePath: relPath.replace(/\.mdx$/, ''),
          readingTime: Math.max(1, Math.ceil(wordCount / 500)),
        })
      }
    }
  }

  scan(CONTENT_DIR)
  return graphs.sort(
    (a, b) => new Date(b.date).getTime() - new Date(a.date).getTime()
  )
}

/**
 * 解析单个 MDX 文件，提取 Mermaid / Explain / Prompt 三个块
 */
export async function getGraphBySlug(category: string, slug: string) {
  const filePath = path.join(CONTENT_DIR, category, `${slug}.mdx`)
  if (!fs.existsSync(filePath)) return null

  const raw = fs.readFileSync(filePath, 'utf-8')
  const { data, content } = matter(raw)

  // 提取 <Mermaid>...</Mermaid> 块
  const mermaidMatch = content.match(/<Mermaid>([\s\S]*?)<\/Mermaid>/)
  const mermaidCode = mermaidMatch ? mermaidMatch[1].trim() : ''

  // 提取 <Explain>...</Explain> 块
  const explainMatch = content.match(/<Explain>([\s\S]*?)<\/Explain>/)
  const explainContent = explainMatch ? explainMatch[1].trim() : ''

  // 提取 <Prompt>...</Prompt> 块
  const promptMatch = content.match(/<Prompt>([\s\S]*?)<\/Prompt>/)
  const promptContent = promptMatch ? promptMatch[1].trim() : ''

  return {
    frontmatter: data as GraphFrontmatter,
    mermaidCode,
    explainContent,
    promptContent,
  }
}

/**
 * 获取所有分类及其图谱数量
 */
export function getCategories(): CategoryInfo[] {
  const graphs = getAllGraphs()
  const categoryMap = new Map<string, number>()
  
  graphs.forEach((g) => {
    categoryMap.set(g.category, (categoryMap.get(g.category) || 0) + 1)
  })

  const categoryMeta: Record<string, { name: string; description: string; icon: string }> = {
    'llm-basics': { name: 'LLM 基础', description: '大语言模型核心架构', icon: '🧠' },
    'rag': { name: 'RAG', description: '检索增强生成', icon: '🔍' },
    'agent': { name: 'Agent', description: '智能体架构', icon: '🤖' },
    'mlops': { name: 'MLOps', description: '模型运维工程', icon: '⚙️' },
    'prompt-engineering': { name: 'Prompt 工程', description: '提示词设计', icon: '✨' },
    'fine-tuning': { name: '微调', description: '模型微调技术', icon: '🔧' },
  }

  return Array.from(categoryMap.entries()).map(([slug, count]) => ({
    slug: slug as any,
    count,
    ...(categoryMeta[slug] || { name: slug, description: '', icon: '📊' }),
  }))
}

/**
 * SSG 用：生成所有路由参数
 */
export function getAllGraphSlugs(): { category: string; slug: string }[] {
  return getAllGraphs().map((g) => {
    const parts = g.filePath.split('/')
    return {
      category: parts[0],
      slug: parts[parts.length - 1],
    }
  })
}
```

```typescript
// components/shared/SearchDialog.tsx
'use client'

import { useState, useEffect, useMemo } from 'react'
import {
  CommandDialog,
  CommandInput,
  CommandList,
  CommandEmpty,
  CommandGroup,
  CommandItem,
} from '@/components/ui/command'
import { useRouter } from 'next/navigation'

import type { GraphMeta } from '@/lib/types'

interface SearchDialogProps {
  graphs: GraphMeta[]  // 构建时传入所有图谱元数据
}

export function SearchDialog({ graphs }: SearchDialogProps) {
  const [open, setOpen] = useState(false)
  const [query, setQuery] = useState('')
  const router = useRouter()

  // ⌘K / Ctrl+K 快捷键
  useEffect(() => {
    const handler = (e: KeyboardEvent) => {
      if ((e.metaKey || e.ctrlKey) && e.key === 'k') {
        e.preventDefault()
        setOpen((prev) => !prev)
      }
    }
    document.addEventListener('keydown', handler)
    return () => document.removeEventListener('keydown', handler)
  }, [])

  const results = useMemo(() => {
    if (!query) return graphs.slice(0, 10)
    const q = query.toLowerCase()
    return graphs.filter(
      (g) =>
        g.title.toLowerCase().includes(q) ||
        g.description.toLowerCase().includes(q) ||
        g.tags.some((t) => t.toLowerCase().includes(q))
    ).slice(0, 15)
  }, [query, graphs])

  return (
    <>
      <button
        onClick={() => setOpen(true)}
        className="flex items-center gap-2 px-3 py-1.5 text-sm text-muted-foreground border rounded-lg hover:bg-muted transition-colors"
      >
        <Search className="h-4 w-4" />
        <span>搜索图谱…</span>
        <kbd className="ml-4 text-xs bg-muted px-1.5 py-0.5 rounded">⌘K</kbd>
      </button>

      <CommandDialog open={open} onOpenChange={setOpen}>
        <CommandInput
          placeholder="搜索架构图…"
          value={query}
          onValueChange={setQuery}
        />
        <CommandList>
          <CommandEmpty>未找到相关图谱</CommandEmpty>
          <CommandGroup heading="架构图谱">
            {results.map((g) => (
              <CommandItem
                key={g.filePath}
                value={g.title}
                onSelect={() => {
                  router.push(`/graph/${g.filePath}`)

                  <p className="font-medium">{g.title}</p>
                  <p className="text-xs text-muted-foreground line-clamp-1">
                    {g.description}
                  </p>
                </div>
              </CommandItem>
            ))}
          </CommandGroup>
        </CommandList>
      </CommandDialog>
    </>
  )
}
```

## 六、验收标准（Acceptance Criteria）

> 每条 AC 均标注优先级：**P0** = 必须达标方可上线，**P1** = 上线首周内修复，**P2** = 可延后。

### 模块 A：图表渲染体验

| AC | 描述 | 优先级 | 验证方法 |
|----|------|--------|----------|
| A.1 | **主题联动**：切换 Dark/Light 模式时，Mermaid 图表节点背景、边框、连线、文字颜色在 300ms 内无缝切换，不出现"黑底黑字"或"白底白字" | P0 | 手动切换 5 次，截图对比 |
| A.2 | **画布缩放**：鼠标滚轮 / 触控板双指可平滑缩放（0.3x - 3x），无闪烁抖动 | P0 | Chrome + Safari 测试 |
| A.3 | **画布拖拽**：长按左键可拖拽平移图表，移动端支持单指拖拽 | P0 | 桌面 + 移动端测试 |
| A.4 | **复位按钮**：画布右下角"恢复 100% 居中"按钮单击即复位 | P0 | 缩放后点击验证 |
| A.5 | **超大图表**：50+ 节点的图表不崩坏布局，内容完整渲染，超出部分通过拖拽查看 | P0 | 用最复杂的测试 MDX 验证 |

## 八、构建时校验脚本

```typescript
// scripts/validate-content.ts
// 运行: npx tsx scripts/validate-content.ts

import fs from 'fs'
import path from 'path'
import matter from 'gray-matter'

const CONTENT_DIR = path.join(process.cwd(), 'content')
const REQUIRED_FIELDS = ['title', 'slug', 'description', 'category', 'diagram_type', 'tags', 'date']

let errors = 0
let total = 0

function validate(dir: string) {
  const entries = fs.readdirSync(dir, { withFileTypes: true })
  
  for (const entry of entries) {
    const fullPath = path.join(dir, entry.name)
    
    if (entry.isDirectory()) {
      validate(fullPath)
      continue
    }
    
    if (!entry.name.endsWith('.mdx')) continue
    total++
    
    const relativePath = path.relative(CONTENT_DIR, fullPath)
    const raw = fs.readFileSync(fullPath, 'utf-8')
    const { data, content } = matter(raw)
    
    // 检查必填字段
    for (const field of REQUIRED_FIELDS) {
      if (!data[field]) {
        console.error(`❌ [${relativePath}] 缺少必填字段: ${field}`)
        errors++
      }
    }
    
    // 检查 <Mermaid> 块存在
    if (!content.includes('<Mermaid>') || !content.includes('</Mermaid>')) {
      console.error(`❌ [${relativePath}] 缺少 <Mermaid> 代码块`)
      errors++
    }
    
    // 检查 <Explain> 块存在
    if (!content.includes('<Explain>') || !content.includes('</Explain>')) {
      console.error(`❌ [${relativePath}] 缺少 <Explain> 解析块`)
      errors++
    }
    
    // 检查 slug 与文件名一致
    const expectedSlug = entry.name.replace('.mdx', '')
    if (data.slug !== expectedSlug) {
      console.error(`❌ [${relativePath}] slug "${data.slug}" 与文件名 "${expectedSlug}" 不一致`)
      errors++
    }
    
    // 检查 category 与目录名一致
    const dirName = path.basename(path.dirname(fullPath))
    if (data.category !== dirName) {
      console.error(`❌ [${relativePath}] category "${data.category}" 与目录名 "${dirName}" 不一致`)
      errors++
    }
    
    if (errors === 0) {
      console.log(`✅ [${relativePath}]`)
    }
  }
}

console.log('🔍 正在校验 /content 目录...\n')
validate(CONTENT_DIR)

console.log(`\n📊 结果: ${total} 个文件, ${errors} 个错误`)
if (errors > 0) {
  console.error('\n❌ 校验失败，请修复上述错误后再构建。')
  process.exit(1)
} else {
  console.log('\n✅ 全部校验通过！')
}
```

在 `package.json` 中添加：

```json
{
  "scripts": {
    "validate": "tsx scripts/validate-content.ts",
    "prebuild": "npm run validate",
    "build": "next build",
    "dev": "next dev"
  }
}
```
